## Travel Agency Frontend (travel-frontend)

Клиентска част (публичен сайт) за туристическата агенция – листинг на екскурзии, страници за дестинации, категории, SEO оптимизация и др. Изграден с Next.js (pages router), MongoDB/Mongoose и styled-components/Tailwind в компонентите.

### Стартиране (development)

```bash
npm install
npm run dev
```

По подразбиране фронтендът работи на `http://localhost:3001`.

Необходими са коректни променливи в `travel-frontend/.env` (MongoDB, S3 и др.), вкл. `MONGODB_URI` към базата `travel`.

### Как се показват екскурзиите (overview)

- Екскурзиите се съхраняват в колекцията `products` (модел `Product`), но моделът вече представлява **екскурзии/пътувания** (trip), а не книги.
- Списъците с екскурзии се визуализират в:
  - `pages/trips.js` – основен списък с филтър и търсене.
  - `pages/trip/[id].js` – детайлна страница за една екскурзия.
  - `pages/destinations.js` и `pages/destination/[name].js` – страници по дестинации.
  - `pages/categories.js` и `pages/category/[id].js` – страници по категории.
- Данните идват от същата база, която се използва от админ панела (`travel-backend`).

### Процес на създаване на екскурзии (обобщено, от гледна точка на целия проект)

1. **Създаване на категории (в админ панела)**
   - В `travel-backend` → страница `Категории`:
     - Създай родителски категории (напр. „Почивки“, „Екскурзии“, „City break“).
     - (По желание) добави **properties** (напр. тип хотел, изхранване и др.).
     - (По желание) създай подкатегории, свързани с родител.

2. **Създаване на екскурзии (в админ панела)**
   - В `travel-backend` → страница `Екскурзии` → `Добави нова екскурзия`:
     - Попълни основна информация (име, държава, град, дати, цена, места и др.).
     - Избери категория – ще се покажат properties от категорията и нейните родители.
     - Попълни стойности за properties и качи снимки.
     - Запази.

3. **Визуализация във фронтенда**
   - След създаване в админ панела, екскурзиите автоматично се показват:
     - на началната страница (`pages/index.js`) – секции „featured“ и „new trips“;
     - в списъка с екскурзии (`/trips`);
     - в съответните страници за дестинации и категории.
